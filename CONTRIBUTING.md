# Contributing to Typehue

First off, thank you for considering contributing to Typehue! It's people like you that make Typehue a better tool for everyone.

## Code of Conduct

By participating in this project, you are expected to uphold our Code of Conduct (standard professional behavior).

## How Can I Contribute?

### Reporting Bugs

- **Check if the bug has already been reported** by searching on GitHub under [Issues](https://github.com/ammar-sultan-89/typehue/issues).
- If you can't find an open issue addressing the problem, [open a new one](https://github.com/ammar-sultan-89/typehue/issues/new). Be sure to include a title and clear description, as much relevant information as possible, and a code sample or an executable test case demonstrating the expected behavior that is not occurring.

### Suggesting Enhancements

- **Check if the enhancement has already been suggested.**
- Open a new issue and describe the feature you would like to see, why it would be useful, and how it should work.

### Your First Code Contribution

1. Fork the repository.
2. Clone your fork: `git clone https://github.com/ammar-sultan-89/typehue.git`
3. Create a branch for your changes: `git checkout -b my-feature-branch`
4. Install dependencies: `npm install`
5. Make your changes and ensure they follow the project style.
6. Run linting: `npm run lint`
7. Commit your changes: `git commit -m "Add some feature"`
8. Push to your fork: `git push origin my-feature-branch`
9. Submit a pull request!

## Development Setup

Typehue is built with React, Vite, and Tailwind CSS.

- **Node.js**: Ensure you have Node.js 18+ installed.
- **Package Manager**: Use `npm`.

```bash
# Install dependencies
npm install

# Start development server
npm run dev

# Build for production
npm run build
```

## Pull Request Guidelines

- Keep pull requests small and focused on a single change.
- Ensure the code builds and passes linting.
- Update documentation if you are adding new features.
- Link the PR to any related issues.

Thank you for your contributions!
